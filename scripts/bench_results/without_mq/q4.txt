## Creating a New Integration Package Using the LangChain CLI

To create a new integration package using the LangChain CLI, use the `langchain integration new` command:

### Installation

```bash
pip install langchain-cli
```

### Basic Usage

```bash
langchain integration new --name my-integration
```

This will:
1. Prompt you for the integration name (e.g., `my-integration`)
2. Prompt you for the PascalCase name (e.g., `MyIntegration`) for class naming
3. Create a new folder `langchain-my-integration/` with the integration template
4. Automatically run `uv sync --dev` to install dependencies (if uv is installed)

### Options

- `--name`: The name of the integration (kebab-case, e.g., `my-integration`)
- `--name-class`: The PascalCase name for classes (e.g., `MyIntegration`)
- `--src` / `--dst`: Copy specific template files to custom destinations

### Additional Command

The CLI also provides `langchain integration create-doc` to create documentation for your integration components (ChatModel, Tool, VectorStore, Embeddings, etc.).

The generated package includes:
- Chat models, embeddings, LLMs, document loaders, tools, retrievers, vectorstores, and toolkit templates
- Unit and integration test scaffolding
- Standard project structure following LangChain conventions
