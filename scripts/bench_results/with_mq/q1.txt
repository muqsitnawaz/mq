The commit standards for the LangChain monorepo are:

## Commit Standards

The repository uses **Conventional Commits** format for PR titles. Key requirements:

1. **Format**: `type(scope): description`
2. **Lowercase**: All commit/PR titles should be lowercase (except proper nouns/named entities)
3. **Scope required**: All PR titles MUST include a scope - no exceptions

### Examples:
```txt
feat(langchain): add new chat completion feature
fix(core): resolve type hinting issue in vector store
chore(anthropic): update infrastructure dependencies
```

For allowed types and scopes, refer to `.github/workflows/pr_lint`.
