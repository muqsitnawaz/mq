Based on my search through the documentation, the LangChain CLI does **not** have a direct command to create a new integration package. Here's what I found:

## Available CLI Commands

The `langchain-cli` provides these commands:
- `langchain app new` - Create a new LangServe application
- `langchain app add` - Add templates to an existing app
- `langchain template new` - Creates a new **template** package (not integration package)

## Creating Integration Packages

The documentation shows an `integration_template` folder exists at `libs/cli/langchain_cli/integration_template/` which serves as a reference structure, but there's no CLI command to scaffold it automatically.

To create a new integration package, you would need to:

1. **Install the CLI**: `pip install -U langchain-cli`
2. **Manually copy** the integration template structure from `libs/cli/langchain_cli/integration_template/`
3. **Replace placeholders** like `__package_name__`, `__ModuleName__`, and `__module_name__` with your actual values
4. The template includes boilerplate for Chat Models, Embeddings, and LLMs

Alternatively, use `langchain template new NAME` to create a template package (different from an integration package).
