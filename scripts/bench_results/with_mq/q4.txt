The `integration` command is registered in the CLI. Here's the answer:

---

To create a new integration package using the LangChain CLI, use:

```bash
langchain integration new --name <integration-name>
```

**Options:**

| Option | Description |
|--------|-------------|
| `--name` | The name of the integration (e.g., `my-integration`). Required, prompts if not provided. |
| `--name-class` | PascalCase name for classes (e.g., `MyIntegration`). Used for class names like `MyIntegrationVectorStore`. |
| `--src` | Template file to copy (can be used multiple times) |
| `--dst` | Destination path for the new file |

**Example:**

```bash
langchain integration new --name parrot-link --name-class ParrotLink
```

This creates a new integration package with the standard structure including chat models, embeddings, and LLM boilerplate code. The template includes placeholder classes that you can customize for your integration.
